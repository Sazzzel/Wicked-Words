## User Experience (UX)  

### **User Stories**

### Primary Goal

The website will be designed as an easy to use word game for all ages to enjoy.
The website will be interactive and from a mobile first design. 
The website should look spooky and give off a Halloween vibe without being too scary.

### **First Time Users**

* New users will find a spooky yet friendly looking website.
* New users will find it easy to play and navigate the site to find out how to play.
* New users will be able to use the website on any device.
  
### **Returning Users** 

* Returning users can easily find the mode that they prefered last time and try to beat their previous score.
* Returning users should not have to enter their name again if on the same computer and web browser (until they clear their browser cookies)

# Creation Process  
  
### **1. Strategy** 

* We need a website that is friendly and inviting to all ages.
* We want it to be animated and easy to use. 
* Easy to choose the game mode and the level.
* A Halloween theme to sync with the time of year.

### **2. Scope**  

* The website needs to be available in multiple sizes for different devices.
* Easy to access the game style and difficulty levels.
* Find out about the developers in about us.
* How to play the game information.

### **3. Structural**  

The website will be simple to use and easy to navigate for all ages. 
The website will use a canvas as a backdrop and as the game screen.
Modals will be used to get infortion to the player.


### **4. Skeleton** 

This website will be a one page site.
The canvas will prvide a moving backdrop behind the site.
The game will play after the game mode and level have been selected.

## Deployment

GitHub was used to deploy the website. These were the steps taken to acheive this:

1. Login to GitHub account.
2. Navigate to the project repository, scoops_pp1.
3. Click the Settings button near the top of the page.
4. In the left-hand menu, find and click on the Pages button.
5. In the Source section, choose 'main' from the drop-down, select branch menu.
6. Select 'root' from the drop-down folder menu.
7. Click 'Save' and after a few moments the project will have been made live and a link is visible at the top of the page.
